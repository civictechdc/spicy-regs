Spicy Regs MCP Server

A regulations.gov mirror, queryable from your AI tools.

A remote Model Context Protocol server that exposes the Spicy Regs regulatory dataset — dockets, documents, and public comments — to any MCP-compatible client. It runs DuckDB queries against a public Cloudflare R2 parquet bucket; no API key, no local install required.

Endpoint https://mcp.spicy-regs.dev/mcp

Set up in your AI tool

Pick your client below. Most accept the remote HTTP endpoint directly; for local-process clients there's a uvx stdio option too.

Claude.ai web & desktop

Requires a Pro, Max, Team, or Enterprise plan.

  1. Open Settings → Connectors → Add custom connector.
  2. Name it Spicy Regs.
  3. Set the URL to https://mcp.spicy-regs.dev/mcp.
  4. Leave authentication as None (the dataset is public).
  5. Save, then toggle the connector on in any conversation.

Claude Code

Two transports work — pick one.

Remote HTTP:

claude mcp add --transport http spicy-regs https://mcp.spicy-regs.dev/mcp

Local stdio via uvx (no remote dependency):

claude mcp add spicy-regs -- uvx --from "spicy-regs @ git+https://github.com/civictechdc/spicy-regs" spicy-regs-mcp

Cursor

Add to your MCP config, then enable it under Settings → MCP.

~/.cursor/mcp.json  (global)  ·  .cursor/mcp.json  (per-project)

{
  "mcpServers": {
    "spicy-regs": {
      "url": "https://mcp.spicy-regs.dev/mcp"
    }
  }
}

VS Code Copilot

Add a workspace MCP config (Agent mode required).

.vscode/mcp.json

{
  "servers": {
    "spicy-regs": {
      "type": "http",
      "url": "https://mcp.spicy-regs.dev/mcp"
    }
  }
}

Windsurf

Add to the Cascade MCP config, then refresh the server list.

~/.codeium/windsurf/mcp_config.json

{
  "mcpServers": {
    "spicy-regs": {
      "serverUrl": "https://mcp.spicy-regs.dev/mcp"
    }
  }
}

Any other MCP client

Most clients accept either a remote streamable-HTTP URL or a local stdio command.

Remote (streamable HTTP):

https://mcp.spicy-regs.dev/mcp

Local (stdio) — runs the published console script with uvx:

uvx --from "spicy-regs @ git+https://github.com/civictechdc/spicy-regs" spicy-regs-mcp

Tools exposed

Three read-only tools over the R2 dataset.

ToolPurpose
list_sourcesList the logical tables in the R2 dataset.
describe_table(table)Get the column schema for one table.
query_sql(sql, max_rows=25)Run a read-only SQL query against the R2 views.

Available views: dockets, documents, comments, comments_index, feed_summary, agency_stats, agency_monthly_volume. Full schemas live in the data dictionary.
